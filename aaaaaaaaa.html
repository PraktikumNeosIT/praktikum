Was ist Git?


  • Git ist eine kostenlose Software.

  • Git wird zur Versionsverwaltung von Dateien benutzt, die Dateien werden in Repositorys (repos) gespeichert.

  • Vor allem im programmier Bereich, wird es benutzt um die Aenderungen zu Ueberwachen.

  • (Versionsverwaltungsprogramme sind Systeme die zur Erfassung von Aenderungen in Dokumenten oder Dateien verwendet werden. Alle Datei Versionen werden mit einem zeit Stempel und Benutzer Erkennung gesichert und koennen spaeter wieder hergestellt werden)

    • Push, pull und commit


  • Push (deutsch: druecken): push ist, wenn man eine Aenderung hochlaedt dann pusht man diese.

  • Pull (deutsch: ziehen): Bei einem pull Ueberprueft er, ob alle Dateien auf aktuellen stand sind und wenn nicht werden die nicht aktuellen Dateien aktualisiert.

  • Commit (deutsch: geaendert): Die Aenderungen werden angeschaut, Ueberprueft und schon mal lokal gespeichert.

  • Branch (deutsch: Zweig): Branching ist, von der Hauptentwicklungslinie abzuweichen und auf einem Nebenzweig (meist eine Kopie) weiter zu arbeiten.

    • Wer nutzt Git?


  • Git wird von grossen und kleinen Unternehmen benutzt.

  • Auch privat Leute benutzen Git da sie immer ihre einzelnen Schritte angucken/verfolgen koennen und auch andere diese einzelnen Schritte nachvollziehen koennen.

    • Was ist ein Repository?


  • Ein Repository ist grob beschrieben eine Datenbank.

  • In einem Repository befinden sich seine gespeicherten Dateien inklusive Ihrer vorherigen Versionen dadurch ist es einfach, den Aufbau zu durchschauen.

  • Es gibt von allen veraenderten Dateien auch noch Klone von den Vorgaengern und den bereits vorhandenen Dateien.
